🔒 NDA Agreement
Slide 2

Legal consent before the survey begins. Captures name, drawn signature, and agreement timestamp.

nda_name
string — Full legal name
e.g. "John Smith"
nda_signature
base64 image — Drawn signature on canvas
nda_agreed
boolean — Did they check the agreement box
nda_timestamp
ISO datetime — When they signed
e.g. "2026-03-28T01:10:55.057+00:00"
4 fields

business_name

business_website

business_stage

revenue_range

location
string | null — City/region
admin_hours
integer | null — Weekly hours on admin work

🗂 Walkthrough Feedback
Slides 6–7

Their reaction to the prototype. Rating each page, leaving notes, pinning things, and picking favorites.

page_ratings
object — Rating (1–5) per page
e.g. { home: 4, sales: 5, marketing: 3, ... }
page_notes
object — Text feedback per page
e.g. { home: "Love the dashboard layout", ... }
pins
array — Pinned locations on the prototype
favorites
string[] — Which pages they starred
e.g. ["schedule", "members"]
4 fields
📊 Engagement Tracking
Passive

Behavioral data collected automatically — how long they spend on each page and where they click in the prototype.

page_timestamps
object — Seconds spent per prototype page
e.g. { home: 38, sales: 10, schedule: 5, ... }
click_data
array — Every click inside the prototype iframe
e.g. [{ x: 12.31, y: 34.29, page: "home", target: "A", timestamp: ... }]
engagement_data
object — Reserved, not yet implemented
Currently always {}

💰 Pricing Signals
Slides 5, 8–10

Willingness to pay — blind price (before seeing product), smart price (after value calc), and final anchor price.

blind_price
integer | null — What they'd pay before seeing the product
Slide 5 — "gut feeling" price
hours_saved
integer | null — Hours/week they think the tool saves
hourly_rate
integer | null — What they value their time at $/hr
smart_price
integer | null — Auto-calculated from hours × rate
Shows them the "value" of the tool before asking final price
final_price
integer | null — What they'd actually pay monthly
The real WTP signal — after seeing product + value calc
added_features
string | null — What features would make them pay more
6 fields
