Where Claude thinks the error is
Root cause diagnosis
▾
📍 booking-page → free-trial-form → submit handler
The most likely cause is a JavaScript event listener conflict on mobile. The submit button uses an onclick handler that fires correctly on desktop but is being blocked on iOS Safari by the 300ms tap delay or a touchstart / touchend conflict in the GHL embed wrapper.

Secondary possibility: the form's required field validation is failing silently — a field is empty or has an invalid value but the error state isn't being shown on mobile viewports due to a CSS overflow or z-index issue hiding the validation message.

Third possibility: the GHL calendar embed inside the form is blocking the submit button on smaller screens — the calendar overlay div has a higher z-index than the submit button, making the button untappable.

Proposed steps to fix it
Ordered by likelihood · check these first
▾
1
Open the booking page on iOS Safari (not Chrome) and inspect the submit button tap area. If nothing happens on tap, confirm with DevTools remote debugging whether the click event is firing at all.
2
Check for z-index stacking issues — open the GHL page editor, inspect the section containing the form, and confirm no overlapping elements are sitting above the submit button on mobile breakpoints.
3
Test all required fields — fill the form completely with valid data on mobile and retry. If it submits, the issue is hidden validation UI. Add explicit error styling for mobile.
4
If using a custom HTML form embedded in GHL — add touch-action: manipulation to the submit button CSS to eliminate the 300ms delay on iOS.
5
Last resort — swap the submit handler from onclick to a form.addEventListener('submit') listener which is more reliable cross-platform.
