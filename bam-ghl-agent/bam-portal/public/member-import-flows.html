03 Take over billing — the core constraint

The portal can create a new sub but cannot cancel a foreign (CoachIQ/GHL/dashboard) sub — Stripe Connect blocks writes to subs it didn't create. So every change leaves a manual cancel.

Double-bill guard: the new sub is anchored to the old sub's next-charge date, and the cancel becomes a 🔴 urgent due-dated Action Item + a red "N old subs still need cancelling" banner in Members & Finish. The banner counts open action items → drops as staff complete them.

Why the paste is manual: CoachIQ has no API to set a sub-id, and the webhook only sends {{user.id}} — never the profile id its admin URL uses. So "CoachIQ ↗" can only land on the academy's client list, and the sub-id paste is a human step at Finish.

05 Known gaps & edge cases

From an independent gap-scan of the flow. The high ones are real bugs; most trace back to one root cause (below).

Root cause behind most highs: three out-of-sync "portal-owned" markers — metadata.origin (take-over/onboarding), metadata.source (setup-monthly), and two different PORTAL_OWNED_ORIGINS lists (webhook ≠ members.js). A member whose sub uses source instead of origin falls through activation, badge, and management gates. Standardize the marker → fixes #1, #2, #3.
🔴 High · bug

setup-monthly subs never activate

Sub stamps metadata.source, the webhook checks metadata.origin → member stuck payment_method_required forever after they pay. setup-monthly.js:192

🔴 High · bug

No-sub dated pause → wrongly resurrected

pause-date-fix inserts a pause row; the cron Phase B flips status paused→live at pause_end even with no sub → a "live" member billing nothing. members.js:1450

🔴 High · bug

verify-cancel ≠ action item done

Cancelling the old sub greens the take-over row but doesn't complete the Action Item or clear the subs_to_cancel banner — they're decoupled. take-over.js:136

🔴 High · bug

"✓ set up" badge lies

Checks only live + ghl_contact_id, not whether the portal can actually manage the sub → a foreign-sub member shows "billing on the portal" falsely. client-portal.html:18955

🟡 Med

Commitment terms lost

Take-over / change-sub bill month-to-month with no commitment record → a mid-commitment member loses the remaining term.

🟡 Med

CoachIQ invite writes nothing

"Supposed to be on it" only copies text — no invited_at, no nudge if they never sign up; member lingers ⏳ waiting.

🟡 Med · multi-tenant

GTA-hardcoded billing

Change-plan (PLAN_TO_PRICE), currency "cad", HST 1.13 hardcoded → breaks for non-Ontario / non-GTA academies.

🟡 Med

coachiq-sync omits the un-linked

Finish CoachIQ list only shows linked members → "invited, never signed up" members vanish from the post-billing paste step with no warning.

🟡 Med

Promote drops rows silently

Duplicates / nameless rows are skipped; take-over's bulk promote fire-and-forgets the skip list → nothing tells the academy who didn't make it.

🟡 Med

GHL step ordering

GHL link (step 4) reads live members but runs before promote → can find "0 to link"; members promoted later never get GHL-linked in the pass.

⚪ Low

Re-import duplicates

Re-running take-over makes a new sub each time; cancel Action Items + cancellation rows have no dedup → pile up on re-runs.

⚪ Low

cancel-at-period-end invisible

A foreign sub set to cancel-at-period-end still bills once more, but staff get no "will bill once more on DATE" surfacing.

Standardizing the portal-owned marker is the single highest-leverage fix — it clears 3 of the 4 high bugs at once.
