+ Pros

  • Most screen real estate per client
  • Shareable URLs per client (deep links)
  • Tabs cleanly separate concerns (setup vs activity)

− Cons

  • Slowest to switch between clients (back/forward navigation)
  • Tabs hide functionality you might want to see at once
  • Most code to build (routing, multiple tabbed views)

My recommendation: A (list + side panel)

You said you want one place to view AND edit. A keeps editing in flow (no modal pop-out). Switching clients is 1 click. Activity timeline + integrations fit on one screen. Fastest for the daily-use case of "scan clients, fix one thing on each."

B is fine if you do deep work on one client at a time. C is overkill for what we're building unless you're going to add a LOT more per-client data later (call logs, tasks, leads, etc. embedded inline).
