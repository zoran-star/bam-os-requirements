The Unified Offer

Today the academy onboarding has a section for Training, a section for Teams, and the build menu has separate items for Youth Academy, Camps/Clinics, and Internal Tournaments. Most of these ask the same questions in different words. This spec collapses them all into one thing — an Offer — with a small set of variations that only differ by pricing model, time horizon, and a few flavor-specific fields.

Core insight

Everything an academy sells is an Offer. Classes, teams, camps, leagues, tournaments, gym rentals — same skeleton, different flavor. The "Type" field picks which fields show. Every offer collects its info in 5 sections: General Info · Value · Pricing · Sales · Onboarding.

When an academy first lands in the portal, a tracker sits in the top-right of the screen with 6 circle-checkbox sections: Meta Ads · Website · Branding · Offers · Staff · Locations. Each is clickable and jumps the user to that section. Each section has an "I'm done here" button so the user can declare completion (not every field is mandatory, so we need explicit done-ness).

What this does to the "Build something new" menu

Today the build menu has 10 tiles. Some are actually offer-builders (Youth Academy, Camps, Internal Tournament). Those move under the Offers card on Business Blueprint. The menu shrinks and stays focused on ad-hoc build requests that aren't about selling something to athletes.

Build menu item todayWhat it becomesWhy
Youth Academy Offer type: Training Same data shape as a class. "Academy" is marketing positioning, not a different structure — folded in.
Camps / Clinics Offer type: Camp / Clinic Same. Start/end dates instead of recurring schedule. Single-day = clinic, multi-day = camp.
Internal Tournament Offer type: Tournament One-off paid event. Brackets and registration windows are type-specific extras.
Gym Rental Offer type: Gym Rental Confirmed as an offer. B2B sales motion (different buyer) but the same data shape — pricing tiers, agreements, notifications, etc.
Upsells Linkage in Sales section Each offer's Sales section has an Upsells builder — pick other offers to point at + description. Standalone "Upsells" build flow deprecates.
Player Intake Stays in build menu It's a form/process, not something the academy sells.
New Hire Stays in build menu HR workflow, not an offer.
Staff Member Stays in build menu Setup for a staff member's profile and access, not an offer.
Sponsor Inquiry Stays in build menu B2B sponsorship intake — different business model, different sales motion.
Promo Stays in build menu It's a marketing campaign / discount, not an offer.
Class Offer type: Training "Class" is just an offer with the Training type. Onboarding's separate Class flow goes away.

After this shake-out the build menu is: Player Intake, New Hire, Staff Member, Sponsor Inquiry, Promo, + the existing "Build something else" overflow. The 6 offer-types (Training, Team, Camp/Clinic, League, Tournament, Gym Rental) get configured under Business Blueprint > Offers instead.
