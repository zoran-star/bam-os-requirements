The Unified Offer

Today the academy onboarding has a section for Training, a section for Teams, and the build menu has separate items for Youth Academy, Camps/Clinics, and Internal Tournaments. Most of these ask the same questions in different words. This spec collapses them all into one thing — an Offer — with a small set of variations that only differ by pricing model, time horizon, and a few flavor-specific fields.

Core insight

Everything an academy sells to an athlete or parent is an Offer. Classes, teams, camps, leagues, tournaments — same shape, different flavor.

Pricing and add-ons are the differentiators. Every offer has them. Every offer also has the same skeleton: who it's for, when it runs, how to sign up, who gets notified, what the registration form asks, and what assets go with it.

Offer variations

Six flavors. Every flavor has the common skeleton plus a few extras. The "Type" field on an offer picks which extras show.

Type: training

Training

Any ongoing skill-development program with a recurring schedule. Covers everything from a single drop-in class to a full multi-pillar youth academy — same data shape, different marketing.

Type-specific
  • Capacity (per slot)
  • Closed dates
  • Recurring weekly schedule
  • "Program structure" long-form (used by academy-style offers)
Type: team

Team

Competitive team. Has tryouts, a season, leagues, and tournaments. Players commit to the whole season.

Type-specific
  • Team level (Local / AAU / Elite)
  • Season dates
  • Tryouts (yes/no → dates, cost, capacity, what to bring)
  • Competition (leagues + tournaments)
Type: camp_clinic

Camps / Clinics

Short-term intensive. Fixed start/end dates. A clinic is just a one-day camp. Sign up once.

Type-specific
  • Start and end dates (same day = clinic)
  • Schedule / format (per-day breakdown)
  • Bring-with list
Type: league

Internal League

Recurring in-house league. Athletes register for a season. Brackets and standings tracked.

Type-specific
  • Season start / end
  • Registration window
  • Brackets / divisions
  • Individual vs. team signup
Type: tournament

Internal Tournament

One-off competitive event. Brackets, rules, registration deadline. Sign up once.

Type-specific
  • Event dates
  • Brackets (yes/no → describe)
  • Individual signups allowed
  • Rules
Type: gym_rental

Gym Rental

Selling court time to outside groups, teams, or events. Per-hour pricing, often B2B. Different buyer than the athlete-side offers, same overall data shape.

Type-specific
  • Per-hour rate + minimum block
  • Court availability windows
  • Insurance / waiver requirements
  • Setup / teardown time

Scheduling — by offer type

Every offer has a Schedule section, but the questions inside it change based on Type. A camp asks "start and end dates"; a class asks "weekly schedule"; a gym rental asks "available booking windows". Same section, conditional fields.

Training schedule

Recurring weekly

Same blocks repeat every week (with break windows). Athletes show up to their slot.

Fields
  • Schedule consistency: Y/N
  • Weekly blocks: day · start · end · location (builder)
  • Capacity per slot
  • Closed dates / break windows
  • Year-round or seasonal?
  • If irregular → how is the schedule communicated?
Team schedule

Season + tryouts + games

Practice is recurring during the season. Game schedule usually comes from an external league.

Fields
  • Season start / end dates
  • Tryouts: Y/N → date(s), time(s), location, capacity
  • Practice schedule (weekly builder, during season)
  • League name(s) — where games come from
  • Tournament commitments (rough count)
  • Game calendar link (external)
  • Off-season activity (optional)
Camps / Clinics schedule

Single day OR multi-day intensive

Fixed dates. One day = clinic. Multiple days = camp. Daily timetable usually repeats.

Fields
  • Start date / end date
  • Days included (e.g. Mon-Fri vs M/W/F)
  • Daily timeline same every day? Y/N
  • If yes → start / end time + lunch break
  • If no → per-day schedule builder
  • Sessions-per-day (single / double-session AM+PM)
  • Registration deadline
League schedule

Recurring season

Like a team but in-house. Games on a recurring day; full season has registration → games → playoffs.

Fields
  • Season start / end
  • Game day(s) + time window (e.g. Sat 10am-2pm)
  • Number of weeks
  • Registration opens / closes
  • Playoff week(s)
  • Awards / banquet date (optional)
Tournament schedule

Single event (1-3 days)

One-time event. Game schedule generated after registration closes.

Fields
  • Event date(s) — single day or weekend range
  • Check-in time
  • First game time / estimated end time
  • Registration deadline
  • Schedule generation: pre-set OR generated post-reg
  • Awards / closing time
Rental schedule

On-demand booking windows

Customer books a block within the gym's available windows. No fixed event date.

Fields
  • Available windows (builder: day-of-week · start · end)
  • Minimum block (hours)
  • Maximum block (hours)
  • Buffer between rentals (minutes)
  • Advance notice required
  • Blackout dates / unavailable periods
  • Booking method: real-time calendar / request / phone

Shared across all types: Location (pick from the Settings > Locations list), staff/coaches assigned, "who notify when schedule changes".

What this does to the "Build something new" menu

Today the build menu has 10 tiles. Some are actually offer-builders (Youth Academy, Camps, Internal Tournament). Those move under the Offers card on Business Blueprint. The menu shrinks and stays focused on ad-hoc build requests that aren't about selling something to athletes.

Build menu item todayWhat it becomesWhy
Youth Academy Offer type: Training Same data shape as a class. "Academy" is marketing positioning, not a different structure — folded in.
Camps / Clinics Offer type: Camp / Clinic Same. Start/end dates instead of recurring schedule. Single-day = clinic, multi-day = camp.
Internal Tournament Offer type: Tournament One-off paid event. Brackets and registration windows are type-specific extras.
Gym Rental Offer type: Gym Rental Confirmed as an offer. B2B sales motion (different buyer) but the same data shape — pricing tiers, agreements, notifications, etc.
Upsells Folds into Add-ons Add-ons IS the upsell mechanism now. The "Upsells" build flow likely deprecates.
Player Intake Stays in build menu It's a form/process, not something the academy sells.
New Hire Stays in build menu HR workflow, not an offer.
Staff Member Stays in build menu Setup for a staff member's profile and access, not an offer.
Sponsor Inquiry Stays in build menu B2B sponsorship intake — different business model, different sales motion.
Promo Stays in build menu It's a marketing campaign / discount, not an offer.
Class Offer type: Training "Class" is just an offer with the Training type. Onboarding's separate Class flow goes away.

After this shake-out the build menu is: Player Intake, New Hire, Staff Member, Sponsor Inquiry, Promo, + the existing "Build something else" overflow. The 6 offer-types (Training, Team, Camp/Clinic, League, Tournament, Gym Rental) get configured under Business Blueprint > Offers instead.
