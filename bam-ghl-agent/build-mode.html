Where Claude thinks the error is
Most likely: JavaScript event listener conflict on mobile. The submit button onclick is blocked on iOS Safari by the 300ms tap delay or a touchstart/touchend conflict in the GHL embed wrapper.

Secondary: Required field validation failing silently — error state not visible on mobile viewports.

Third: GHL calendar embed inside the form is blocking the submit button on smaller screens via z-index.
🛠
Fix steps
1. Open booking page on iOS Safari and confirm tap event firing
2. Check z-index stacking in GHL editor for mobile breakpoints
3. Test with all fields filled — check for hidden validation errors
4. Add touch-action: manipulation to submit button CSS
5. Replace onclick with form.addEventListener('submit')
📦
CSS fix to apply
.submit-btn {
  touch-action: manipulation;
  -webkit-tap-highlight-color: transparent;
  position: relative;
  z-index: 10;
