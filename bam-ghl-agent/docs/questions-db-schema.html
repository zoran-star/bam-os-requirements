Two Relationship Patterns

// Same-page conditional
Question A (Check One)
  ↓ Dependent On = A.id
  ↓ Dependent On Value = ['Yes']
Question B (shows if A = Yes)
// Builder sub-fields
Block Builder Q
  ↓ Parent Question = Q.id
  ↓ sort_order = 1, 2, 3…
Sub-field (no Places Asked)

Hard Rules

One question per menu item. Split with menu-item-specific wording. Shared generics are rare.
Yes/No follow-ups → always conditional. Wire via Dependent On + Dependent On Value. No exceptions.
Sub-fields never carry Places Asked. They inherit from their parent.
Question text is the natural key. Unique across the whole table.
Options only on Check/Dropdown. Required when Input Type is one of those three.
Page & Mandatory never null. Integer and boolean respectively.
