Training the Sales Agents

Three AI texting agents (booking, confirm, closing) draft messages to leads. Humans review every draft. Every human correction becomes a lesson that trains the bot. Here is the whole loop, start to finish.

2 STORE — the lesson library
one table, two kinds of lesson
🗄️agent_lessons table
One row per lesson in Supabase (the database). Each row carries: which agent, which scope, which academy (client_id), and whether it's active.
A spreadsheet of everything the bots have been taught.
→
📍Academy lesson
client_id = this academy, scope = 'academy'. Only loads for that one academy's bot.
"OUR gym does it this way." Stays with that gym.
→
🌐General lesson
client_id = NULL, scope = 'general', tagged preset = free_trial. Loads for EVERY academy running that playbook.
"Good selling works everywhere." Shared with all gyms.

Right now every academy runs the same playbook: the free-trial preset (get the lead to book a free trial). When we add other sales styles later, the preset tag is how we keep their lessons separate.

every new draft reads the library ↓↓
3 USE IT LIVE — lessons steer the next draft
the bot re-reads what it was taught, every time
🔎loadConfig pulls lessons
Before drafting, the agent queries this academy's lessons + the shared general set, filtered to its own agent and active=true.
"What have I been taught that applies here?"
→
🧩Assemble the prompt
Lessons drop into a <learned_lessons> block alongside the facts (price, schedule), examples, and per-contact memory.
The bot's instructions = rules + facts + past corrections + who this person is.
→
✨Smarter draft
Same lead situation, better reply. The mistake you corrected doesn't come back.
It stops making the mistake you fixed.
Facts vs. habits live on different shelves. Prices and schedules go in editable Knowledge sections; corrections go in lessons. Why it matters: a lesson is injected as "OVERRIDE the rules above," so if a price ever got saved as a lesson it would beat the real price forever. Facts stay facts; lessons only carry behavior.
the pile grows... time to tidy ↓↓
4 CLEAN UP — the /consolidate-lessons skill (the librarian)
runs at 15+ raw lessons, or every 2 weeks
📥Dump the pile
lessons-io.mjs dump pulls the raw lessons (plus the draft-vs-sent evidence) for one academy.
Lay every taught lesson on the table.
→
🧠Cluster & route
Claude + you merge duplicates, then sort each lesson 4 ways ↓
Tidy, dedupe, and file each one correctly.
↓
📘→ Brain FACT
It's a fact (price, hours, coach) → update the Knowledge section, don't keep as a lesson.
📍→ Academy lesson
Behavior only this gym needs → keep local.
🌐→ General lesson
Selling craft every gym benefits from → promote to the shared brain (preset-tagged).
🗑️→ Drop
Stale or already covered → archive it.
A longer prompt is a dumber bot. Why cleanup IS the training: every extra lesson glued to the prompt slightly weakens how well the bot follows ALL of them ("context rot"). Consolidation must NET-REDUCE the pile - the old raw rows get set active=false so the live prompt stays lean. Running the librarian isn't housekeeping - it's the training.
↓
⛏️Mine for onboarding gaps
For each academy lesson ask: "which fact, collected at signup, would have prevented this correction?" → a candidate in the intake ledger.
If we had to teach it, we should ASK every future gym up front.
→
🏗️Build it into the V2 UI
Recommend a home (BB card / offer setup / onboarding page / Settings), workshop it with you, then build the question and add the Notion data-point.
The next gym gets asked automatically - their bot starts smarter.

The human review gate (Hawkeye) is the engine, not training wheels - it's the only place real ground-truth corrections come from. We loosen it to auto-send per academy only once its correction rate drops, never all at once.
