# Vercel deployments — business folder

## Production
The repo root deploys to Vercel project **`bam-os-requirements`** under `zoran-stars-projects`. The whole repo root deploys, with rewrites configured in `/vercel.json` at the repo root:

| Path | Destination |
|---|---|
| `/summary` | `/business/business/summary.html` (investor summary) |
| `/playbook` | `/fullcontrol-investor-playbook.html` |
| `/investor` | `/prototypes/fc-company/index.html` |

To edit the live investor summary: edit `business/business/summary.html`, push to `main`, Vercel auto-deploys.

## Branch previews
Every pushed branch gets a Vercel preview URL of the form:

```
bam-os-requirements-git-<branch-slug>-<hash>-zoran-stars-projects.vercel.app
```

Examples from past PRs:
- `claude/objective-franklin-bfcb30` → `bam-os-requirements-git-claude-obje-b62779-zoran-stars-projects.vercel.app`
- `claude/nostalgic-cerf-cd4498` → `bam-os-requirements-git-claude-nost-736f9a-zoran-stars-projects.vercel.app`

The slug + hash are generated by Vercel and not predictable from the branch name. After pushing a branch, check GitHub PR comments (vercel[bot]) or the Vercel dashboard to find the exact URL.

## Gotcha: free-tier deploy cap
Vercel free tier limits to **100 deployments per day** across the org. PR #8 (April 23 2026) hit this:
> Resource is limited - try again in 24 hours (more than 100, code: "api-deployments-free-per-day").

Heavy edit days that push every small change can blow through the cap. Mitigations:
- Batch edits per push when iterating on copy
- Avoid pushing trivial whitespace changes
- Wait 24h or upgrade to Pro if the cap blocks an investor demo
